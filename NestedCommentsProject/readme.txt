Работа с приложением начинается со StartPage.php.На данной странице присутствует форма для добавления комментариев 0 уровня.В файле присутствует валидация полей формы,не допускающая пустых строк а также строк,состоящих из одних пробелов.Функции Add(Delete...)Comments формируют соответствующие строки для ajax-запросов к CommentsHandler.php.функция NestedCommentsHandler отвечает за показ\скрытие вложенных комментариев.Функция Handler формирует ajax-запрос,а также изменение содержимого стартовой страницы после получения ответа.На странице присутствует функция Comment_Output,отвечающая за вывод всех комментариев.Наличие этой функции необходимо для обновления при перезагрузке статической стартовой страницы в случае изменения БД и отсутствия ajax-запроса,а также подгрузки актуальной БД при первом запуске.Если в подобных функциях нет необходимости,этот метод можно спокойно убрать(В таком случае необходимость в подключении к БД также отпадает).

CommentsHandler.php
Обрабатывает запрос с параметрами: Тип запроса(q = {I,D,R,S}),Имя пользователя(username) и текст комментария(comment_text).Функция DeleteNestedComments рекурсивно удаляет вложенные комментарии,проверяя наличие ссылок на текущий комментарий.Функция Comment_Output рекурсивно выводит комментарии,проверяя ссылки и присваивая вложенным комментариям класс NestedComments,который используется для показа\скрытия вложенных комментариев.

Вложенные комментарии считываются с помощью alert.Я не стал использовать форму,дабы не загромождать страницу.Можно добавлять идетичные комментарии.Я добавил кнопки к каждому комментарию,т.к посчитал,что это будет самый простой вариант.На верстку времени практически не тратил,т.к Вы указали,что придираться к ней никто не будет.
